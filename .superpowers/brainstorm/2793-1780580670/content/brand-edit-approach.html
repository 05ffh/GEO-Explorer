品牌编辑页 — 哪种交互方式更合适？

编辑品牌名称、别名、行业。现有 API PUT /api/brands/{id} 已就绪。

A

独立编辑页

独立页面 /brands/{id}/edit，全页表单，提交后跳回品牌 Dashboard。适合字段较多、需要专注编辑的场景。

B

Dashboard 内联编辑

在品牌 Dashboard 顶部放一个「编辑」按钮，点击后 HTMX 将静态信息替换为编辑表单，保存后原地刷新。轻量，不离开工作上下文。

C

侧边抽屉

点击编辑按钮后在右侧滑入一个抽屉面板，内含编辑表单。不跳页，不挤占主内容区，适合作为辅助操作。

推荐: B — Dashboard 内联编辑

理由: Brand 模型只有 3 个可编辑字段 (name/aliases/industry)，表单极简。内联编辑零跳转、零页面切换，符合 HTMX 设计哲学。Edit/View 切换是前端状态，不需要路由。
